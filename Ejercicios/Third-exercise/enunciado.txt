# Ejercicio 3

En este directorio hay un chatbot con inteligencia artificial que corre
completamente en tu máquina, sin usar ninguna API externa. Tiene tres servicios:

  - llama:    servidor de IA (Ollama con el modelo qwen2:0.5b)
  - backend:  API en Flask que recibe mensajes y consulta a llama
  - frontend: interfaz web servida con Nginx

El docker-compose.yml ya tiene la estructura básica. Tu tarea es completarlo
con tres cosas: el archivo .env, los healthchecks y el orden de arranque.

## 1. Archivo .env

Docker Compose puede leer variables de un archivo .env en el mismo directorio.
Fijate que el compose ya usa ${MODEL} para pasarle el modelo al backend.

Creá un archivo .env con el siguiente contenido:

    MODEL=qwen2:0.5b

## 2. Healthchecks

Un healthcheck le dice a Docker cómo saber si un servicio está realmente listo,
no solo corriendo. Se define así:

    healthcheck:
      test: ["CMD", "comando", "argumento"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 10s

Agregá healthchecks a los servicios llama y backend:

  llama:
    Ejecuta el comando "ollama list" dentro del contenedor.
    Usá un start_period de 20s porque tarda en iniciar.

  backend:
    Hace un curl a http://localhost:5000/api/health

## 3. Orden de arranque

Ahora que los servicios tienen healthchecks, podés usarlos para garantizar
el orden de arranque con depends_on extendido:

    depends_on:
      nombre-del-servicio:
        condition: service_healthy

  - backend debe esperar a que llama esté sano
  - frontend debe esperar a que backend esté sano

## 4. Restart policy

Agregá restart: unless-stopped a los tres servicios para que se reinicien
automáticamente si fallan.

## Comandos útiles

Construir las imágenes (hacelo antes de levantar, tarda unos minutos):
    docker compose build

Levantar los servicios:
    docker compose up

Ver los logs de un servicio:
    docker compose logs backend

Ver el estado y healthchecks de los contenedores:
    docker compose ps

Bajar los servicios:
    docker compose down
